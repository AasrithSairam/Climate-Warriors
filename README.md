# Climate Warriors — AI-Powered Clinical Intelligence Platform

## What is this

Climate Warriors is an AI-agent-powered platform that establishes a persistent, privacy-preserving patient memory layer. It acts as a secure, digital "health backpack" that contains a patient's entire longitudinal record, allowing orchestrated AI agents to synthesize clinical insights at the point of care.

## Why we built this

Healthcare data is often siloed and inaccessible, leading to fragmented care and medical errors. Our goal is to empower patients with total control over their medical history while providing doctors with instant, specialty-specific clinical summaries generated by a network of AI agents.

## Repository structure

- **src/** — Core components of the platform
  - **backend/** — Node.js/Express API with Prisma and SQLite
  - **frontend/** — React/Vite dashboard with glassmorphism UI
  - **ai/** — Python/FastAPI orchestration layer for AI agents
- **challenges/** — Challenge descriptions and acceptance criteria
- **datasets/** — Sample medical records and ingestion scripts
- **docs/** — Project documentation and onboarding materials
- **scripts/** — Helper scripts for setup and execution
- **.github/** — Issue/PR templates and CI workflows

## Dockerization

We provide a simple, opinionated Docker setup to build and run the app in both development and production modes.

### Files added:

- **Dockerfile** — Production-ready multi-stage image.
- **docker-compose.yml** — Developer-friendly compose file to run the app and services locally.

### Build and run (Docker)

```bash
# build image
docker build -t climate-warriors:latest .

# run container
docker run --rm -p 3000:3000 --env-file .env climate-warriors:latest
```

### Using Docker Compose (development)

```bash
# start services (rebuild if needed)
docker-compose up --build

# stop
docker-compose down
```

## Dockerfile notes

The provided Dockerfile is a minimal multi-stage example. It handles the frontend build and serves the backend. For the AI layer, ensure you have your `GEMINI_API_KEY` configured in the environment.

## Prerequisites

- Docker >= 20.x

## Quick start

1. Fork this repo and clone it locally.
2. Add your team name and project details in this README top section.
3. Implement your solution in `src/` and provide challenge acceptance tests in `challenges/`.
4. Run the demo locally with `scripts/run_demo.sh` or `docker-compose up`.

## Development

Use `scripts/setup.sh` to install language dependencies for local development.
Mount `src/` into containers in `docker-compose.yml` for live reload during development.

## Testing

Put automated tests in `src/ai/test_agents.py` or a dedicated `tests/` directory. CI will run tests automatically.

## Security

Do not commit secrets. Use `.env` and add `.env` to `.gitignore`.
